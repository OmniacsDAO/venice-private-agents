How It Works

The privacy guarantee: Venice AI processes all data with no-data-retention inference. Prompts and completions are never stored, logged, or used for training. Raw analysis data exists only in transit — it enters Venice, analysis comes out, and zero trace remains.

  1. Agent fetches public on-chain data from Ethereum (Etherscan) + Base (Blockscout) — balances, token activity, governance, TVL
  2. Data + optional private context sent to Venice AI for confidential analysis
  3. Venice returns a split output: public verdict (safe to publish) + private report (detailed findings)
  4. Public verdict is returned directly in the API response — severity level + one-sentence summary
  5. Full report gets stored via x402-pastebin — plain text, retrievable by token
  6. Optionally, report is time-locked via x402-timecapsule for delayed disclosure
  7. Every action is an x402 USDC micropayment on Base — auditable, transparent, immutable

Analysis Types

Treasury — Analyze wallet holdings, token concentrations, spending patterns. Verdict reveals health status; report details positions.

Governance — Analyze DAO proposals, voting patterns, delegate power. Verdict reveals participation health; report details centralization risks.

Risk — Analyze protocol risk factors: TVL, audits, dependency chains. Verdict reveals risk level; report details exposures.

Due Diligence — Research a project or address for legitimacy signals. Verdict reveals trust level; report details red flags.

Privacy Architecture

  • Venice AI — No-data-retention inference. Prompts/completions never stored.
  • Private context — Sent to Venice only, never persisted anywhere in our stack.
  • Public verdicts — Contain only conclusions. No raw data, no reasoning exposed.
  • Full reports — Stored on x402-pastebin as plain text, retrievable by token.
  • Time capsules — Content sealed until future date. Inaccessible before unlock.
  • Payment receipts — Every x402 payment is a USDC transfer on Base. Auditable but content-opaque.
